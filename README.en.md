## 👨‍💻 About me

I've been working with data for over **13 years**. I started pulling network cables at a public school, became a SQL Server DBA, lived through the on-premises BI era, made it to cloud, Lakehouse, Databricks, and today I work as an **AI Engineer** building AI agents in production.

Beyond the technical work, I'm active in the Brazilian data community: Speaker at Microsoft AI Tour, SQL Saturday, Data&AI and Azure Bootcamp, and author of the blog [datainaction.dev](https://www.datainaction.dev/).

I believe that:

- **Good AI is born from well-architected data** — without a foundation, agents don't scale.
- **Code is still a critical asset**, even when AI is the one writing it.
- **Software Engineering fundamentals** (DDD, TDD, SDD, bounded context) became **more relevant**, not less, in the agent era.

I don't believe in "best cloud" or "best language." I believe in choosing well for the context and **executing with quality**.

---

## 🛠️ What I'm building right now

I work at [**Belake.ai**](https://www.belake.ai/) ([Dataside](https://dataside.com.br/)) — an AI agent platform with **native Text2SQL** for Databricks, Snowflake, Redshift, BigQuery, SQL Server, PostgreSQL, Fabric and Synapse, with native MCP integrations and LangGraph orchestration.

Day to day:

- 🧠 End-to-end AI agent engineering (architecture, orchestration, deployment, observability)
- ⚙️ Backend in **TypeScript/Node**, frontend in **React + Vite**, AI layer in **Python**
- 🧪 **TDD** and **SDD** as the foundation of the AI-assisted development cycle
- 🤖 Coding with **Claude Code**, **Cursor** and **Codex** in my daily workflow

**Currently exploring:** advanced multi-agent orchestration patterns, evals for production agentic systems, and hexagonal architecture applied to AI workflows.
